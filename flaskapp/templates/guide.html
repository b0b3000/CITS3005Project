Ontology Implementation Guide

Types (Classes)

  • Procedure
  • Step
  • Toolbox
  • Tool
  • Item
  • Part
  • Image

Each class represents a specific entity within the ontology, such as a Procedure, Step, Toolbox, etc.

Object Properties (Relations)

  • has_step: Connects a Procedure to its Step.
  • has_tool: Connects a Toolbox to its Tool.
  • has_image: Connects a Step to its Image.
  • has_part: Connects a Procedure to its Part.
  • part_of: Connects Item or Part to another Item (transitive).
  • used_in: Connects a Tool to the Step it is used in.
  • has_item: Connects a Procedure to an Item.
  • has_toolbox: Connects a Procedure to its Toolbox.
  • subprocedure: Connects one Procedure to another (transitive).

Data Properties (Attributes)

  • step_number: An integer representing the step number in a Step.
  • step_description: A string describing a Step.
  • has_name: A string representing the name of a Procedure, Tool, Item, or Part.

Property Constraints

The following constraints apply to various types within the ontology:

  • Procedure: Must have at least one Step, exactly one Toolbox, one Item, one Part, and one name.
  • Part, Item, Tool: Must have exactly one name.
  • Step: Must have exactly one step_number and one step_description.

SWRL Rules (Logical Enforcements)

The following rules ensure logical consistency:

  • Rule 1: Ensures that if a Procedure has a Step, then the step must be of type Step.
  • Rule 2: Ensures that a Procedure having a Toolbox implies that the toolbox must be of type Toolbox.
  • Rule 3: Ensures that a Procedure having an Item implies that the item must be of type Item.
  • Rule 4: Ensures that a Procedure having a Part implies that the part must be of type Part.
  • Rule 5: Ensures that a Toolbox having a Tool implies that the tool must be of type Tool.
  • Rule 6: Ensures that a Step having an Image implies that the image must be of type Image.
  • Rule 7: Ensures that a Tool used in a Step implies that the step must be of type Step.
  • Rule 8: Ensures that a Part that is part of an Item must imply that the item is of type Item.
  • Rule 9: Ensures that if a Part is part of another Item, the second item must also be of type Item.

SWRL Additions (Advanced Rules)

Additional rules handle specific cases:

  • Rule A1: If two procedures have the same item, they are subprocedures of each other.
  • Rule A2: If an item in one procedure is a part of an item in another procedure, the procedures are subprocedures of each other.
  • Rule A3: If a part in one procedure is part of an item in another procedure, the procedures are subprocedures.
  • Rule A4: Adds transitivity to subprocedures.
  • Rule A5: Ensures that tools used in steps of procedures must appear in the toolbox of the procedure.
  • Rule A6: Ensures transitivity of the part_of relation between Part and Item.
  • Rule A7: Ensures transitivity of the part_of relation between Items.
